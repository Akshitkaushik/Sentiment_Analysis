:Store API Credentials;
note right
Save login details like ID and password securely
in the Automation Anywhere Credential Vault.
This ensures protection and eliminates visibility
in the code. The bot will retrieve these details
at runtime.
end note

:Store API Endpoints;
note right
Save all API web addresses in a secure file or
the Credential Vault. This allows the bot to
access them securely and dynamically.
end note

:Read Input Data;
note right
Process records or files step by step from the
defined input source, such as an Excel file
or database.
end note

:Make Initial API Call;
note right
Use the Automation Anywhere REST Web Service
Action to connect to APIs. Fetch details like URL,
credentials, and data from the vault or file, and
save the API response.
end note

:Extract API Token;
note right
Extract and save the token (if provided) from
the API response for use in the next API calls.
end note

:Make Next API Call;
note right
Use the extracted token for authentication.
Send the next request using secure information
and validate the response.
end note

:Update Records;
note right
Based on the API response, update results in
an organized manner, like saving data to an
Excel file or database.
end note

:Process Next Record;
note right
Move to the next record and repeat the process
until all data is handled.
end note


stop
@enduml
